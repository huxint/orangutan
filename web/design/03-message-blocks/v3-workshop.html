“ 查一下 Anthropic 文档里 prompt-caching 的 TTL 上限——我怀疑 ephemeral 的 5 分钟上限在实战里会导致很多缓存 miss。

Anthropic 官方：prompt-caching 使用 cache_control: {"type":"ephemeral"}，默认 5 分钟 TTL，到期后必须重写缓存块。实际命中率由请求到达率决定——

math 预期命中率 DollarBlock → Block · 规范化后 · KaTeX
P(hit) = 𝔼 [ exp(−λ · Δt) ] · (1 − ε)

其中 λ 是请求到达率（/s），Δt 是连续请求间隔，ε 是缓存失效概率（服务端偶尔 evict）。设 λ = 0.3/s, TTL = 300s，预期 hit 率 ≈ 0.998。

import math

def expected_hit_rate(lam: float, ttl: float) -> float:
    """Fraction of calls that land inside the ephemeral window."""
    return 1 - math.exp(-lam * ttl)

assert expected_hit_rate(0.3, 300) > 0.99
