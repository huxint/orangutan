# References

Curated external references and distilled summaries of upstream material that we
need to consult repeatedly.

Good candidates:

- Framework or deployment references that we rely on (kept up to date in-repo so
  agents don't need network access).
- Design system / API usage notes.
- Compact summaries of external standards or partner integrations.
- Distilled audits of legacy code we are replacing.

Avoid dumping large vendor manuals here. Keep references curated and scoped.

## Files

- [`orangutan-legacy-audit.md`](orangutan-legacy-audit.md) — what we learned from the
  legacy `orangutan/` codebase and what we deliberately replace.
- [`harness-template-distill.md`](harness-template-distill.md) — the harness-template
  scaffold's philosophy in one place.
- [`third-party-libs.md`](third-party-libs.md) — notes about the libraries we use
  beyond what's captured in [`../rules/libraries.md`](../rules/libraries.md).
